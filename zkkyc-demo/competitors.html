🇪🇺 欧盟 eIDAS 2.0 / EUDI Wallet
政府推动
欧盟正在推的数字身份钱包，2026-2027 强制落地。政府签发凭证→公民存手机钱包→选择性披露。政策层面最大的行业催化剂。
  • ✅ 法律强制：所有欧盟成员国必须支持
  • ✅ 覆盖 4.5 亿人口
  • ⚠️ 不一定用 ZK（可能是传统选择性披露）
  • ⚠️ 进度慢，官僚体系效率低
Microsoft Entra Verified ID
企业级
微软的企业级去中心化身份方案。公司签发员工/供应商凭证，跨组织验证。主打 B2B 场景。
  • ✅ 企业客户基础庞大（Azure AD 集成）
  • ✅ W3C DID / VC 标准
  • ⚠️ 不针对 crypto/DeFi 场景
  • ⚠️ 无 ZK 证明，选择性披露有限

💡 差异化定位

✅ 我们的优势

1. 完整的 7 步端到端演示 — 市面上最直观的 zkKYC 教育工具 2. 同时覆盖技术视角 + 用户视角 + 商业视角，适合不同受众 3. BBS+ 签名 + Merkle 撤销 + Nullifier 不可关联，三大隐私特性齐全 4. 中英双语，面向国际受众 5. 开源 + 一键部署，任何人可以体验

⚠️ 与生产级的差距

1. 密码学为模拟实现，非生产级安全 2. 无真实 KYC 数据源对接（OCR、活体检测） 3. 未接入真实区块链（链上注册表为内存模拟） 4. 无移动端钱包 SDK 5. 无多发行方 / 跨链互操作

🚀 从 Demo 到产品的最短路径

1. 接入 Polygon ID 基础设施（替换自研密码学） 2. 对接 zkMe 或自建 KYC 入口 3. 部署链上 Registry 合约（Polygon / Ethereum） 4. 开发移动端钱包（存储凭证 + 生成证明） 5. 找一个 DeFi 协议做首个验证方集成
